✅ 方法①：マウスホイールで手動スクロール（最も簡単）

    • Obsidianのプレビューモード or ライブプレビューで

✅ 方法②：キーボードでスクロール操作

キー操作 動作
↑ / ↓ 1行ずつ上下に移動
PageUp / PageDown 1画面分スクロール
Home / End 最上部 / 最下部へジャンプ

📌 タイミングを見て PageDown でパッパッと送る演出もあり。

✅ 方法③：自動スクロール（手放しで録画するなら）

🛠️️ 1. フリーソフトで自動スクロール

ソフト名 特徴
AutoHotKey スクリプトでスクロール制御。最強だけどやや上級者向け
Auto Scroll 直感的なUIでスクロール速度指定できる（Windows向け）
SmoothScroll 全体的にマウススクロールを滑らかにするツール（制御というより演出向け）

🧪 2. ブラウザで再生する形にして、JavaScriptでオートスクロールさせる方法もあるけど、Obsidian内で直接動かすのは難しい

③ AutoScrollを起動
④ OBSで「録画開始」ボタンを押す

⑥ 必要に応じて Ctrl + S で停止、または手動でスクロール

✅ Obsidianの仕様

機能 対応
手動スクロール（マウス・キーボード） ✅ あり
プラグインでの自動スクロール △（可能だが特殊）
標準機能としての自動スクロール ❌ なし

🎯 だからどうするの？

外部ツール（Auto ScrollやAutoHotkeyなど）でObsidianウィンドウを自動でスクロールさせるのが現実的な方法です。

🔧 外部ツールができること

ツール できること
Auto Scroll（旧Skrommel版、またはGitHub版） 対象ウィンドウを上下に自動でゆっくりスクロール
AutoHotkey キーボード操作を定期的に送る（例：5秒ごとにPageDown）
マウスシミュレータ（例：Move Mouse） マウスホイール操作を定期的に模倣する

✅ つまり結論：

Obsidian側にはスクロールの自動化機能はない。
→ 外部ツールで「Obsidianのウィンドウ」を対象に自動操作させることで対応します。

項目 内容
名前 Autoscroll
開発者 Petr Nazarov
目的 Obsidian内で自動スクロール機能を使えるようにする

🛠️ インストール手順（Obsidian内）

1. Obsidian を開く

2. ⚙️「設定（Settings）」→ コミュニティプラグイン（Community plugins）

3. 「安全モード（Safe mode）」をオフ

4. 「プラグインを閲覧（Browse）」

5. 検索欄に Autoscroll と入力

6. Autoscroll を見つけたらクリック → インストール（Install）

7. → 有効化（Enable）

🎮 基本の使い方

📋 コマンドパレット（Ctrl + P）から操作：

コマンド名 内容
Start Autoscroll 自動スクロール開始
Stop Autoscroll 停止
Toggle Autoscroll 開始・停止の切り替え
Increase Speed スクロール速度を上げる
Decrease Speed スクロール速度を下げる

📝 最初は「Start Autoscroll」だけで動作確認してOKです！

⌨️ ホットキー設定（任意）

1. 設定 → ホットキー（Hotkeys）

2. 「Autoscroll」で検索

3. 各機能に好きなキーを割り当て可能（例：F5でスタート、F6でストップ）

⚙️ カスタマイズ設定（Settings内）

1. 設定 → プラグイン → Autoscroll

2. 調整できる内容：

設定項目 内容
Speed スクロール速度（ms単位）
Step size 1ステップで動かす距離（px）
Delay 最初の待ち時間（ms）
Loop 最後まで行ったらまた上から開始するかどうか
Direction 上向き or 下向き

🎯 動画収録で使うなら？

    • 🎥 OBSと一緒に使うときは：

      1. Obsidianで「リーディングモード」にしておく

      2. Autoscrollで「Start」＋速度調整

      3. OBSの「録画開始」で静かに始まる淡々スクロール動画に！

📘 OBS保存先、録画時間上限は？

🙂 タケ：
OBSは録画終了後はどこに保存されるの？


🤖 チャットさん：
いい質問です！
OBS Studioで録画した動画ファイルは、初期設定だと以下の場所に自動保存されます👇

✅ デフォルトの保存場所（Windowsの場合）

📁 C:\Users\あなたのユーザー名\Videos
　＝「ビデオ」フォルダ内

たとえば：

C:\Users\takeshi\Videos

✅ 保存されるファイル名

録画時刻に基づいて自動命名されます：

🗂️ 例：2024-06-19 12-45-23.mp4

✅ 形式

    • デフォルト：mp4

    • 他にも mkv や flv などに変更可能（設定で選べる）

🔧 保存場所を変更したいとき

1. OBSの画面右下 → ⚙️「設定」→ 左側の「出力」

2. 録画 タブを開く

3. 「録画ファイルのパス」欄で保存先を指定できる

📝 ワンポイント注意

    • 録画中にPCがクラッシュするとmp4は壊れる可能性あり
      　→ 心配なら .mkv で保存 → 後でOBS内で .mp4 に変換も可能です

🙂 タケ：
OBSの録画時間上限は？


🤖 チャットさん：
OBS Studioには録画時間の上限は基本的にありません！
ただし、実際にはいくつかの現実的な制約があります👇

✅ 理論上の録画時間：無制限

OBS自体には「録画は○時間までしかできない」などの制限は一切ありません。

⚠️ ただし録画時間を制限する現実のボトルネック

制約 内容
💽 保存先の空き容量 フルHD（1080p）で1時間＝約1～3GB（設定による）
🧠 PCのメモリやCPU負荷 長時間録画でメモリが逼迫すると処理落ちやクラッシュの可能性
🧱 録画形式の限界（特にmp4） mp4は途中でクラッシュするとファイル全体が壊れることも。長時間なら .mkv が安心
🔌 バッテリー（ノートPCの場合） 電源未接続での録画は時間制限に直結
💥 Windowsファイルサイズ上限（まれ） FAT32形式では4GB以上のファイルが保存できない（NTFSなら問題なし）

✅ 録画の安定運用ヒント

状況 推奨設定
長時間録画 .mkv形式で保存（後でmp4に変換可）
ファイルが壊れるのが怖い 録画を1時間ごとに区切って保存する習慣を
ディスク容量が不安 解像度やビットレートを落とす（目安：ビットレート5,000～8,000kbpsで十分）

🎯 結論

🎥 理論上は無制限！でも現実には「ディスク容量」「録画形式」「PC性能」がカギです。
安定して録るなら .mkv ＋定期保存＋余裕のある容量で！

✅ 起きていること：クロップ→拡大で画質が劣化する

操作 何が起きてる？
1920×1080で録画 → クロップ（例：中央640pxだけ） 録画解像度はそのままだけど、縮小エリアだけを表示→強制的に拡大表示になる
結果 文字が少し拡大されてピクセルが荒れる（＝輪郭ボケる）

🛠️️ 解決策①：最初から「狭い解像度」で録画する

例：縦スクロール用に 640×1080 や 720×1280 に設定して録画

OBSの設定手順：

1. OBS右下 → ⚙️「設定」→「映像」

2. 解像度を以下に変更：

項目 推奨設定（例）
基準（キャンバス）解像度 720×1280（縦動画用） or 960×1080（中央寄せ）
出力（スケーリング）解像度 同上に合わせる

📝 このサイズなら最初から左右に余白がない状態で録画でき、クロップ不要＆ボケない！

🛠️️ 解決策②：Obsidian側で「フォントサイズ・本文幅」を調整して中央寄せに近づける

    • Settings → Appearance → Font Size を大きめに（20px〜）

    • Settings → Editor → Readable Line Width をオフにすると本文が横いっぱい使える（狭くもできる）

🎯 解決策③（緊急）：OBS側で拡大せず、そのまま録画

    • クロップはするけど拡大はしない

    • 拡大しても画質を保つには「高解像度録画」＋「高ビットレート設定」が必要（重くなる）
