✅ GitHub Pages のブランチ設定手順（最新版UI）

🔸 1. GitHubの自分のリポジトリを開く

（例：https://github.com/あなたのユーザー名/リポジトリ名）

🔸 2. 上部メニューから「⚙ Settings」をクリック

※ 左側の「Code」　「Issues」　「Pull requests」などの並びにあります

🔸 3. 左側メニューの下の方にある「Pages」をクリック

    • 項目名は Pages または Pages settings（古いUIでは「Pages」だけ）

🔸 4. 「Source（ソース）」の項目を探す

    • ドロップダウンで None になっていたら、

        • ✅ main を選択（またはブランチを作成する）

        • ✅ / (root) を選択（フォルダ）

👇 こんな設定にするのが基本：

Source: [main] / [root]

🔸 5. Save をクリック

→ 数秒〜1分でサイトが公開されます！

URLは：https://ユーザー名.github.io/リポジトリ名/

✅ その後：ファイル追加できるようになる！

ブランチ設定が完了すると：

    • Add file ボタンが出てくるようになる

    • index.html をアップすれば、トップページが開けるようになる

🏗 リポジトリを“館”として運営するイメージ

館の名前 リポジトリ名 公開URL（自動で生成される）
A館（技術系） a-kan https://username.github.io/a-kan/
B館（創作系） b-kan https://username.github.io/b-kan/
C館（映像まとめ） c-kan https://username.github.io/c-kan/

✅ 特徴と利点

項目 内容
完全に分離された構造 リポジトリごとにファイルも設定も独立。混ざらない。
URLで見分けやすい /a-kan/, /b-kan/ で直感的に整理できる
管理しやすい それぞれ別の目的・実験・テーマで展開可能
GitHub Pagesでは無料で何個でもリポジトリ作れる ✅ ← ここ重要です

🔁 対応する「エントランス」設計も可能

たとえば、本館（メインリポジトリ）の index.html に、各館へのリンクを並べると完璧！
