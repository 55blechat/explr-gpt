📘 行間気になる？

🙂 タケ：

🤖 チャットさん：
はい、HTML化したファイルの行間（箇条書き部分含む）は、CSSで調整可能です。
VS Code や Obsidian のプレビューでは、ブラウザとは異なるスタイル（CSS）が使われているため、行間が異なって見えるのは自然なことです。

🔧 HTMLの行間を調整する方法（箇条書き含む）

1. CSSで line-height を指定

例：

ul, ol {
  line-height: 1.4; /* 数字を変えれば調整可能（例: 1.2 や 1.6など） */
}

2. liタグ単体に適用してもOK

li {
  line-height: 1.4;
}

3. 調整範囲の参考値

    • 1.0: 詰まった印象（狭い）

    • 1.4〜1.6: 標準的で読みやすい

    • 2.0以上: ゆったり（特に縦スクロール動画向け）

✅ 適用例（あなたのテンプレートで調整する場合）

<style>
  body {
    line-height: 1.6; /* 本文全体の行間 */
  }

  ul, ol {
    line-height: 1.4; /* 箇条書きだけ少し詰める、もしくはゆるめる */
  }
</style>

必要であれば、master_template.html に合わせて適切な箇所を提案することもできます。
気が向いたときにでも、行間の理想的な印象（詰めたい・ゆるめたい）を教えてもらえれば調整案を出しますよ。

❗でもまだ「詰まりきってない」理由

👇そのすき間の残りの正体は、実は p の下マージン（margin-bottom）です！

p {
  margin-block-end: 1em;

ulのmargin-topを詰めても、pのmargin-bottomを詰めないと効果が半減してたというわけ。

✅ 対処法（決定版CSS）

.crossnote p {
  margin-block-end: 0.2em;
}

.crossnote p + ul {
  margin-top: 0;
}

これで：

    • p の下マージン（末尾の余白）を詰めて

    • ul の上マージンも 0 にして

    • 全体で1行分もないくらいに自然に詰められます
