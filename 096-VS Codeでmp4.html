✅ 方法1：HTML化して動画埋め込み（王道）

これを .html ファイルにして 右クリック →「ブラウザで開く」 で再生できる。

✅ 結論：MPEでも .mp4 を再生できないことがある主な理由はこれ👇

🔹1. <video> タグがMPEの "Safe Mode" によってブロックされている

MPEはセキュリティのため、デフォルトでHTMLタグの一部を制限してる。
<video> タグも、設定次第で無効化される。

🛠️ 対処法：MPEのセキュリティ設定を変更して <video> を許可

🔧 手順：

  1. Ctrl + Shift + P を押す

  2. Markdown Preview Enhanced: Open Settings を選択

  3. settings.json というファイルが開くので、以下を追記：

"markdown-preview-enhanced.previewTheme": "github-light", // 任意のテーマ
"markdown-preview-enhanced.enableScriptExecution": true

👉 enableScriptExecution: true にすることで、<video> の再生が有効になる。

🧪 確認方法（再テスト）

  1. .md ファイルに以下のように記述：
<video src="./video.mp4" controls width="640"></video>
  1. Ctrl + Shift + P →
    　Markdown Preview Enhanced: Open Preview to the Side を再実行

✅ 補足：標準プレビュー（Locked）のほうが安定していた理由

    • 標準プレビューはVS Code本体のWebViewレンダリングを使っていて、HTMLタグをそのまま解釈

    • MPEはカスタムレンダリング（安全性重視）なので、スクリプト・動画タグがブロックされやすい

    • 特に <script>, <iframe>, <video> などはセキュリティ警戒対象

📘 レスポンシブな画面表示幅

🙂 タケ：
controls width="640"⇒これは640pxで固定なの？端末による画面表示幅の変化には対応なし？


🤖 チャットさん：
いい質問！
controls width="640" という指定は、文字通り「横幅を640ピクセルに固定する」指定だから、
画面の大きさ（端末幅）には対応しない、つまりレスポンシブじゃないんだ。

✅ width="640" の挙動

状況 表示される幅
PC画面が1000px 動画は 640px固定
スマホ画面が500px 動画は はみ出る or 自動縮小される（がコントロールしにくい）

つまり、スマホなどの狭い画面では不都合が出る可能性がある。

✅ レスポンシブ対応したい場合

🔁 ① %指定 に変える

<video src="./sample.mp4" controls width="100%">

    • 横幅を親要素（Markdown本文）の幅に合わせて可変

    • 一番シンプルでおすすめのレスポンシブ対応

💡 ② style 属性で柔軟に

<video src="./sample.mp4" controls style="width: 100%; max-width: 640px;">

    • 画面が大きければ 最大640px

    • 画面が小さければ 親要素の幅にフィット

    • スマホでもPCでも自然な表示

✅ ③ CSSクラスを使う（HTMLファイル化した時用）

Markdownだけで完結させたい場合は上の方法で十分だけど、
HTML変換を前提にするなら、クラスでスタイル指定もアリ：

<video src="./sample.mp4" controls class="responsive-video">

/* HTML内に追加 */
.responsive-video {
  width: 100%;
  max-width: 640px;
}

✅ 結論：Markdown＋動画ならこれでOK！

<video src="./sample.mp4" controls style="width: 100%; max-width: 640px;"></video>

これでスマホでもPCでも無理なく再生可能。
NoteやGitHub Pagesに貼るHTML化にもそのまま流用できるよ。

✅ 対応手順：Locked Previewの内容をHTMLに書き出すには？

✅ 方法①：「Markdown PDF」拡張機能でHTML出力する（一番シンプル）

  1. 拡張機能パネル（左の四角アイコン）を開く

  2. 「Markdown PDF」と検索
    　→ 作者が「yzane」のやつをインストール！

  3. .md ファイルを開く

  4. Ctrl + Shift + P → Markdown PDF: Export (html) を選ぶ！

🔽 出力形式もいろいろある：

    • Export (html)

    • Export (pdf)

    • Export (png) など

✅ 方法②：「ファイル → 名前を付けてHTMLとして保存」手動方式（素朴だけど確実）

  1. Markdownファイルをプレビューしながら Ctrl + Shift + P

  2. 「Markdown: Print current document to HTML」を探す（ない場合は拡張機能が必要）

  3. 保存先を指定 → .html ファイルが作られる

✅ 方法③：「Markdown Preview Enhanced」でも出力可能（こっちは別レンダラ）

すでにMPE（Markdown Preview Enhanced）が入ってるなら：

  1. Ctrl + Shift + P → Markdown Preview Enhanced: Export (html) を選ぶ

  2. HTML, PDF, PNG などが選べる

※ ただし、Locked Previewとは見た目が違うことがあるので注意！
