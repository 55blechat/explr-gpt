✅ まず結論から

はい、Obsidianではファイル名と本文トップの大見出し（# タイトル）が密接に連動する仕組みがあります。
ただし、完全に切り離すことはできませんが、“見た目だけ分離”は可能です。

🧠 背景の仕組み：Obsidian のファイル名・タイトル連動とは？

Obsidianの基本仕様：

要素 内容
.mdファイルのファイル名 ノートの「名前」＝Obsidian内での識別子
.md本文の最初の行 # タイトル 見出しとしてレンダリングされる
自動補完・バックリンク ファイル名で機能する（[[ファイル名]]）
HTML化時 多くのエクスポーターは、# タイトル を <h1> に変換し、それをHTMLのtitleやIDに使うことがある ←これがジャンプ失敗の原因になる

🔥 つまり何が起きてるか

たとえば：

# 001_あらすじ

本文いろいろ...

というMarkdownファイルを sora-intro.html にリネームしても、

    • HTMLタイトルや目次アンカーリンクなどに残るのは 001_あらすじ の名残

    • # 見出し部分がそのまま <h1 id="001_あらすじ"> になり、URLエンコードが混じってジャンプに失敗する

✅ 解決策（切り離すには？）

✔️ 方法1：見出しとファイル名を一致させない（人間的な方法）

    • ファイル名：sora-intro.md

    • 最初の行を変更：

# Soraの基本まとめ（概要）

本文...

→ Obsidian内のファイル名と、見出し（HTMLの <h1>）が分かれる
→ ジャンプやリンクも分離して管理できる

✔️ 方法2：エクスポート後に <h1> の id="..." を明示的に書き直す

<h1 id="sora-intro">Soraの基本まとめ（概要）</h1>

→ 目次リンク側の href="#sora-intro" が機能するようになる

✔️ 方法3：Obsidian PublishやCustom Exportを使ってルールを固定する（中〜上級者向け）

    • obsidian-export CLIなどでテンプレートを使い、
      ファイル名・本文の見出し・出力ファイル名をそれぞれ制御できる

✅ 今あなたがやった「ソース中一括置換でリンク直し」も正解に近い

ただし：

    • ファイル名とidが違ってる

    • エンコードされてる（例：日本語は %E3%81%82 みたいになる）

    • 目次生成がJSでやられてる場合、リンク元が一括置換に反応しない

などの理由で、うまくいかないケースが出てしまうんです。

✨ 最も現実的な対応パターン

状況 対応
ファイル名は整理したい 英数字＋小文字でシンプルに管理（例：sora-intro.md）
見出しは読みやすくしたい # Soraの基本まとめ（概要） ← 人向けに整える
目次ジャンプで混乱したくない idを手動で指定 or 自動化された目次ツールを使う

もしよければ、「このファイルで、見出しとファイル名を切り離して整理したい」という具体例を1つ出してもらえれば、それに合わせてHTML調整済みテンプレ作れますよ。

🤖 チャットさん：
はい、その観察…本当に正確です。

Obsidian内では、ファイル名＝本文タイトルの関係は実質「切れません」。
しかもおっしゃるとおり、ファイル名（＝ノート名）を「正」として、そこから本文を開くという動作をしています。

✅ Obsidianの基本設計：ファイル名ファースト

Obsidianの扱い方 説明
.mdファイルのファイル名 ノートの名前そのもの（UI上もそれで表示）
# 見出し（本文内） Markdownとしての表示上のタイトル。実際の識別には使われない
ノートを開くとき ファイル名で探して開く（[[ファイル名]] もこの動作）
タブ表示・検索・バックリンク すべてファイル名ベースで動作

🧠 なぜ切れないのか？

    • Obsidianは「フォルダ＝保管庫」「ファイル名＝ノートID」
      → ファイル名が実体そのものと扱われる設計

    • 見出し（# タイトル）はあくまで中身の一部
      → ファイル名と異なっていても開けるが、UI上のラベルや参照対象としてはファイル名が基準

🎯 だからこう見える

ノートを開くとき、保管庫のフォルダ構造を確認し、ファイル名を読み取り、そこから本文をロードする
→ これはまさに Obsidian の“データベースではなくフォルダベース”という思想の体現です。

✅ 結論：Obsidian内では…

    • ファイル名＝ノート名（ID）として扱う

    • 見出しは中身に過ぎず、切り離すことはできない

    • その結果、ファイル名を変えるとリンクが壊れたり、ジャンプできなくなったりする

💡 それでも整理したいときの実践的な妥協案

やりたいこと 方法
ファイル名を英数字で整理したい sora-intro.md などにしておき、
見出しは日本語にしたい # Soraの基本まとめ（日本語タイトル） と書く
Obsidian内のリンクも保持したい エイリアス機能を使う（alias: を使うにはカスタムプラグインが必要） or リンクテキストに別名を表示 [タイトル表示](リンク先ファイル)
